1. REST API
	REST (Representational State Transfer)
	API (Application Programming Interface)
merupakan suatu arsitektu yang memandu user melalui
perantara berupa desain antar muka/UI. REST API memiliki 5
arsitektuk yang sering digunakan programmer untuk berinteraksi :
- GET (sebagai read-only)
- POST (untuk membuat resource)
- PUT (untuk update resource)
- PATCH (update resource)
- DELETE (menghapus resource)

3. Handle Architecture
berfungsi untuk menyederhanakkan penerbitan multisaluran. 
dimana cara kerjanya berkaitan dengan Front-end dan Back-end.
CMS monolitik akan bergantung pada arsitektur Front-end dan Back-end.
Back-end sebagai media penyimpanannya, sedangkan front-end sebagai media
tampilannya.
Selanjutnya konten yang telah dibuat dipublikasikan melalui layanan web atau API
yang dirancang untuk mendorong ke perangkat akhir. Konten yang dibuat akan
disediakan dalam format XML atau Json.
Tigas Manfaat Handler Architekture :
- Agnostic : Kebebasan untuk memilih framework front-end
- Scalability : kemampuan ditingkatkan dan tidak terbatas.
- Agility : mengelola konten dan pengembang front-end dengan kecepatan
yang lebih tinggi dan biaya yang minimum.
